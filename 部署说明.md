# 日喀则市东嘎水库工程智慧建管平台——Nginx 静态部署说明

## 部署包内容

- `site/`：可直接由 Nginx 托管的完整纯前端站点。
- `nginx/dongga.conf`：Nginx 站点配置示例。
- `部署说明.md`：本说明文件。

站点已包含全部页面、组件样式、字体、图标、图片和 Mock 数据，不需要 Node.js，也不需要后端接口。

## 部署步骤

1. 将 `site/` 目录完整上传到服务器，例如 `/usr/share/nginx/html/dongga/`。
2. 将 `nginx/dongga.conf` 放入 Nginx 配置目录，例如 `/etc/nginx/conf.d/dongga.conf`。
3. 如果站点目录不同，修改配置中的 `root` 路径。
4. 执行 `nginx -t` 检查配置。
5. 重新加载 Nginx：`nginx -s reload`，或使用服务器对应的服务管理命令。

## 访问路径

- 登录页：`http://服务器地址/`
- 系统首页：`http://服务器地址/dashboard/`

所有业务页面均已导出为独立静态路由，支持在浏览器中直接刷新。

## 更新方式

升级时先备份旧目录，再使用新包中的 `site/` 完整覆盖站点目录，然后重新加载 Nginx。不要只复制 HTML 文件，否则可能造成脚本与样式版本不一致。

## 注意事项

- 当前版本使用 Mock 数据，数据保存在前端脚本中，刷新页面会恢复为预设模拟数据；浏览器本地保存的登录角色等状态仍按现有系统逻辑工作。
- 若使用 HTTPS、域名或非根路径部署，需要在 Nginx 上补充证书或子路径配置；本包默认部署在域名根路径 `/`。
- 建议对 HTML 禁止长期缓存，对 `/_next/static/` 使用长期缓存；示例配置已经包含这两项策略。
